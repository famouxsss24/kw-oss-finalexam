# 🎬 데모 영상 대본 — 일단막아 · DEFENSE FIRST

> 읽으면서 그대로 따라 찍는 나레이션 + 동작 대본.
> **굵게**는 화면에서 실제로 하는 동작, 〔 〕는 채점 기준 짚는 멘트.
> 권장 길이 7~9분. 마우스는 천천히, 각 단계 말하면서.

## 0. 촬영 전 세팅 (영상 시작 전에 미리)

- [ ] EC2에서 `docker compose up -d` 로 컨테이너 2개 실행해 둔 상태
- [ ] 브라우저 탭 2개 준비: ① `http://<EC2-IP>:8501` (앱)  ② GitHub repo 페이지
- [ ] EC2 접속 터미널 1개 열어두기 (docker ps 보여줄 용)
- [ ] 브라우저 개발자도구(F12) → Network 탭 위치 미리 익혀두기
- [ ] 녹화 해상도 1080p, 글자 잘 보이게 브라우저 확대 110~125%

## 1. 인트로 (약 30초)

> **(앱 로그인 화면 또는 제목 띄운 상태)**

"안녕하세요. 오픈소스소프트웨어실습 기말 대체과제, **방위산업 보안 위협 대응 추천 시스템 '일단막아, DEFENSE FIRST'** 발표를 시작하겠습니다.

이 앱은 방위산업 실무자가 **지금 처한 보안 상황을 입력하면**, 위협 위험도를 진단하고 **상황에 맞는 방어 대응책을 추천**해주는 웹 애플리케이션입니다.

기술 스택은 과제 요건에 맞춰 **프론트엔드는 Streamlit, 백엔드는 FastAPI, 실행 환경은 Docker, 배포는 AWS EC2**로 구성했습니다.
〔평가 1·2·8·9 — 스택 전체를 한 번에 언급〕

특히 이 앱의 핵심은 **추천 계산을 프론트엔드가 아니라 반드시 FastAPI 백엔드에서 수행하고, 그 결과를 JSON으로 주고받는다**는 점입니다. 이 부분을 영상에서 직접 보여드리겠습니다."

이렇게 **프론트와 백엔드를 각각 분리된 Docker 컨테이너로 실행**했고, `docker-compose`로 한 번에 띄웠습니다.

〔평가 9 — 외부 접속 동작〕"

## 3. 로그인 · 부서 워크스페이스 (약 50초)

> **(로그인 화면)**

"이 앱은 단순 추천을 넘어 **부서 단위 협업 워크스페이스**로 설계했습니다.
개인 계정이 아니라, 본인이 속한 **부서 코드로 접속**하면 같은 부서 실무자들이 같은 사건을 공유하며 공동 대응하는 구조입니다.

데모용으로 세 개 부서를 미리 등록해 두었습니다. 그중 **방산보안대응팀, DAPA-OPS**로 접속하겠습니다."

> **(부서 코드 칸에 `DAPA-OPS`, 실무자 이름에 본인 이름 입력 → '워크스페이스 접속' 클릭)**

"**'워크스페이스 접속' 버튼을 누르면**, 프론트엔드가 백엔드의 **`/auth/login` API**를 호출합니다.
백엔드는 입력한 부서 코드가 **SQLite 데이터베이스에 등록된 부서인지 검증**하고, 실무자 정보를 등록하거나 조회한 뒤, 부서·실무자 정보를 돌려줍니다.
그 정보로 **세션이 만들어지면서** 부서 워크스페이스 화면으로 넘어갑니다.

왼쪽 사이드바를 보시면 접속한 부서와 실무자, 그리고 연결된 **FastAPI 주소가 표시**됩니다.
〔평가 12 — 차별성: 부서 워크스페이스·협업 컨셉〕"

## 4. 🚨 사고대응 모드 — 핵심 흐름 (약 2분 30초) ★가장 중요

> **(사이드바 '새 진단' 상태에서, 분기 화면)**

"메인 화면에서는 **'지금 어떤 상황이신가요?'** 라고 묻고 두 가지 모드로 나뉩니다.
하나는 **이미 공격·사고가 발생한 경우의 사고대응 모드**, 다른 하나는 **사전에 대비하는 예방점검 모드**입니다. 두 모드는 입력 항목도, 결과의 톤도 다릅니다.

먼저 **사고대응 모드**를 보여드리겠습니다. **'사고대응 모드 시작' 버튼을 클릭**합니다."

> **(클릭 → 위저드 STEP 1로 진입)**

"입력은 한 번에 다 받지 않고, **단계별 위저드 방식**으로 진행됩니다. 상단 진행바로 지금 4단계 중 몇 번째인지 보여줍니다.
〔평가 1 — Streamlit 입력 인터페이스 / 평가 6 — 입력 구성〕

- **STEP 1, 무슨 일이 일어났나요?** — 침해 유형을 고릅니다. 'APT 표적공격'을 선택하겠습니다."

> **(APT 표적공격 선택 → '다음' 클릭)**

"- **STEP 2, 어떤 자산이 영향받았나요?** — 가장 민감한 '무기체계 설계도면'을 선택합니다. 자산 민감도가 높을수록 위험도 점수가 올라갑니다."

> **(무기체계 설계도면 선택 → '다음')**

"- **STEP 3, 현재 피해 상황** — 칩을 눌러 여러 개를 선택할 수 있습니다. '데이터 유출 정황'과 '확산 중'을 선택하겠습니다. 피해 항목이 많을수록 위험도가 가산됩니다."

> **(칩 2개 선택 → '다음')**

"- **STEP 4, 조직 환경** — 조직 규모와 현재 보안 수준을 입력합니다. '중소협력사', 보안 수준은 '낮음'으로 두겠습니다.

이제 추천을 요청하기 전에, **이 요청이 정말 백엔드로 가는지** 직접 보여드리겠습니다.
브라우저 **개발자도구를 열고 Network 탭**을 띄우겠습니다."

> **(F12 → Network 탭 클릭 → 비워두기)**

"이 상태에서 **'분석하기' 버튼을 누르겠습니다.**"

> **('🔍 분석하기' 클릭)**

"보십시오. Network 탭에 **`/recommend` 라는 POST 요청**이 잡힙니다.
이 요청을 클릭해서 내용을 보겠습니다.

- **Request(요청)** 를 보면, 방금 위저드에서 입력한 값들이 **JSON으로 백엔드에 전송**됐습니다. 모드, 침해 유형, 자산, 피해 상황, 조직 환경이 그대로 담겨 있습니다.
- **Response(응답)** 를 보면, 백엔드가 계산한 결과가 **JSON으로 돌아왔습니다.** 위험도 점수, 위험 등급, 위협 분류, 우선순위, 그리고 기술적·관리적·물리적 **대응책 세 가지**가 들어 있습니다.

여기서 중요한 점은, **이 위험도 계산과 대응책 생성은 전부 FastAPI 백엔드의 로직에서 수행**된다는 것입니다. Streamlit은 입력을 모아 보내고 결과를 받아 보여주기만 합니다.
〔평가 3 — Streamlit→FastAPI 실제 요청 / 평가 4 — JSON 반환 — 이 장면이 핵심 증거〕

그리고 이 결과는 동시에 **SQLite 데이터베이스에 '부서 케이스'로 저장**됩니다. 화면 위에 '부서 케이스 몇 번으로 저장되었습니다'라는 메시지가 그 증거입니다."

> **(개발자도구 닫고 결과 리포트 보여주기)**

"화면에는 결과가 **보안 진단 리포트** 형태로 표시됩니다.
- 상단에 **위험도 게이지** — 점수가 높아 빨간색 HIGH로 표시되고 우선순위는 '긴급'입니다. 위험 등급에 따라 색이 초록·노랑·빨강으로 바뀝니다.
- 아래에는 **추천 대응책 카드** — 기술적, 관리적, 물리적 세 가지 유형으로 나뉘어 있습니다.
- 맨 아래 **관련 규제**와, 어떤 엔진으로 추천했는지 표시됩니다.
〔평가 5 — 결과 적절히 출력 / 평가 7 — 논리적 설계〕"

## 6. 부서 공유 · 협업 기능 (약 1분) — 차별성 어필

"이제 이 앱만의 차별점인 **부서 협업 기능**을 보여드리겠습니다.
사이드바에서 **'부서 공유 케이스'로 이동**하겠습니다."

> **('부서 공유 케이스' 클릭)**

"방금 만든 진단들이 **부서 케이스 목록**으로 쌓여 있습니다. 같은 부서 코드로 접속한 실무자라면 누구나 이 목록을 함께 봅니다.
백엔드의 **`/departments/{부서ID}/cases` API**로 부서별 케이스를 불러온 것입니다.

케이스를 하나 선택하면 진단 결과가 다시 표시되고, 오른쪽에서 **공동 대응**을 할 수 있습니다.

- **상태 변경** — '접수'에서 '조치중'으로 바꿔보겠습니다. 이건 **`/cases/{ID}/status` API**를 호출해 **SQLite에 저장된 케이스의 상태를 갱신**합니다."

> **(상태 드롭다운 변경 → '상태 업데이트' 클릭)**

"- **대응 메모** — 'EDR 로그 확보 완료'라고 메모를 남기겠습니다. **`/cases/{ID}/notes` API**로 메모가 **DB에 기록되고**, 누가 언제 남겼는지와 함께 아래에 표시됩니다."

> **(메모 입력 → '메모 저장' 클릭 → 목록에 뜨는 것 확인)**

"이렇게 단순 추천이 아니라, **부서 단위로 사건을 진단하고 상태를 관리하고 대응 이력을 공유하는 실무형 워크스페이스**로 구현했습니다.
〔평가 12 — 개인별 차별성 강력〕"

## 7. (선택) AI 하이브리드 추천 설명 (약 30초)

> ※ ANTHROPIC_API_KEY를 넣고 'AI 분석' 표시가 뜰 때만. 키 없으면 이 챕터 생략하고 '규칙기반 엔진'으로 설명.

"추천 로직에 대해 잠깐 설명드리면, 이 앱은 **하이브리드 방식**입니다.
- 먼저 **규칙 기반 스코어링 엔진**이 자산 민감도, 위협 유형, 피해 범위, 보안 수준을 종합해 위험도 점수를 항상 계산합니다.
- 여기에 **Claude AI를 연결**해, 그 진단 결과를 바탕으로 **상황에 맞는 대응책 문장을 생성**합니다. 결과 하단의 '● AI 분석' 표시가 그것입니다.
- 만약 AI 호출이 실패하거나 키가 없으면 **자동으로 규칙 기반 대응책으로 전환**되어, 데모 중에도 항상 결과가 나옵니다.
〔평가 7 — 논리적 설계 + 가산점〕"
