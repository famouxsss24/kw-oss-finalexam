# 일단막아 · DEFENSE FIRST — 기획서

> 방위산업 실무자를 위한 **보안 위협 대응책 추천 시스템**
> 오픈소스소프트웨어실습 기말 대체과제 (마감 2026-06-17)

---

## 1. 한 줄 소개

사용자가 **지금 처한 보안 상황**을 입력하면, 위협을 진단하고 **맞춤 방어 대응책**을 추천해주는 웹앱.

> "일단, 막아." — 공격받았으면 빠르게 대응책을, 대비하려면 강화 대책을.

---

## 2. 기술 스택 (과제 필수 요건)

| 레이어 | 기술 | 역할 |
|---|---|---|
| 프론트엔드 | **Streamlit** | 입력 위저드 + 결과 리포트 표시 |
| 백엔드 | **FastAPI** | 입력 받아 추천 결과 생성 → JSON 반환 |
| 통신 | **HTTP (requests)** | Streamlit → FastAPI 실제 호출 (★채점 핵심) |
| 저장소 | **SQLite** | 부서별 케이스·상태·대응 메모 저장 |
| 컨테이너 | **Docker + docker-compose** | front/back 컨테이너 분리 실행 |
| 배포 | **AWS EC2 (Learner Lab)** | 외부 접속 가능한 실제 서비스 |

**데이터 흐름 (반드시 구현):**
```
부서 코드 로그인 → 사용자 입력 → Streamlit → (HTTP POST) → FastAPI 추천 처리 → JSON 반환/케이스 저장 → Streamlit 결과 표시
```
⚠️ 추천 계산은 **반드시 FastAPI에서** 수행 (Streamlit에서 직접 계산 금지).

**협업 흐름:**
```
같은 부서 코드로 로그인한 실무자들 → 동일 부서 워크스페이스 접속 → 케이스 목록/상태/대응 메모 공유
```

---

## 3. 핵심 컨셉 — 2갈래 시나리오

첫 화면에서 사용자 상황에 따라 두 모드로 분기한다.

```
            지금 어떤 상황이신가요?

   ┌────────────────────┐   ┌────────────────────┐
   │ 🚨 이미 공격/사고 발생 │   │ 🛡️ 사전에 대비하고 싶음 │
   │   = 사고대응 모드      │   │   = 예방점검 모드       │
   └────────────────────┘   └────────────────────┘
```

두 모드는 **입력 항목·결과 톤·추천 방향이 다르다** → 차별성 강화.

---

## 4. 입력 흐름 (위저드 / 단계별)

진행바(예: `STEP 2 / 4`)와 함께 한 번에 하나씩 묻는다.

### 🚨 사고대응 모드
| STEP | 질문 | 입력 형태 |
|---|---|---|
| 1 | 무슨 일이 일어났나요? (침해 유형) | 카드 선택: 내부자 유출 / 랜섬웨어 / APT 표적공격 / 공급망 침해 / 물리적 침입 / 피싱 |
| 2 | 어떤 자산이 영향받았나요? | 카드 선택: 무기체계 설계도면 / 소스코드 / 인사·계약정보 / 시험데이터 / 네트워크 인프라 |
| 3 | 피해 상황은? | 토글: 데이터 유출 정황 / 시스템 다운 / 확산 중 / 외부 신고 여부 |
| 4 | 조직 환경 | 조직 규모(대기업/중소협력사/연구소) · 현재 보안수준(낮음/보통/높음) |
| → | **[ 🔍 대응책 분석하기 ]** | 추천 요청 버튼 |

→ **결과: 긴급 대응 우선순위 + 즉시 조치 대응책 (긴박한 톤)**

### 🛡️ 예방점검 모드
| STEP | 질문 | 입력 형태 |
|---|---|---|
| 1 | 무엇을 지키고 싶나요? (보호 자산) | 카드 선택 (위와 동일 자산군) |
| 2 | 우려되는 위협은? | 다중선택: 내부자 / APT / 공급망 / 물리 / 랜섬웨어 |
| 3 | 조직 환경 | 조직 규모 · 현재 보안수준 |
| 4 | 규제 준수 대상 | 다중선택: 방위산업기술보호법 / 군사기밀보호 / ISMS / 없음 |
| → | **[ 🔍 위험도 진단하기 ]** | 추천 요청 버튼 |

→ **결과: 위험도 진단 + 강화 대책 로드맵 (차분한 톤)**

## 5. 추천 로직 — 하이브리드 (규칙 + AI)

> 채점은 로직 완성도가 아니라 "결과가 **논리적으로 설계**되었는가"를 봄. 하이브리드로 충분 + 가산점.

```
입력값(JSON)
   │
   ▼
[1] 규칙 기반 스코어링 (FastAPI 내부, 항상 동작)
   - 침해유형 × 자산 민감도 × 피해범위 → 위험도 점수(0~100)
   - 위협 카테고리 분류 + 우선순위 산정
   │
   ▼
[2] Claude API 호출 (선택적 고도화)
   - 규칙 결과를 컨텍스트로 전달 → 상황 맞춤 대응책 문장 생성
   - (기술적 / 관리적 / 물리적 대책)
   │
   ▼ (Claude 키 없음 / 호출 실패 시)
[3] 규칙 기반 대응책 템플릿으로 자동 fallback
   │
   ▼
JSON 응답 → Streamlit
```

→ AWS EC2 데모 중 AI가 실패해도 **fallback으로 항상 결과가 나옴** (데모 안 깨짐).

### FastAPI 응답 JSON 예시 (스키마 초안)
```json
{
  "risk_score": 78,
  "risk_level": "HIGH",
  "threat_category": "APT 표적공격",
  "priority": "긴급",
  "recommendations": [
    { "type": "기술적", "title": "...", "detail": "..." },
    { "type": "관리적", "title": "...", "detail": "..." },
    { "type": "물리적", "title": "...", "detail": "..." }
  ],
  "regulations": ["방위산업기술보호법 제X조"],
  "engine": "ai"
}
```

## 7. 디자인 방향 (간지 / 깔끔)

- **다크 네이비 테마** + 포인트 컬러(시안 `#22d3ee` / 앰버 `#f59e0b`) → 국방·보안 무드
- **위험도 = 색**: 낮음 초록 / 보통 노랑 / 높음 빨강 → 한눈에 읽힘
- **카드 UI + 아이콘**: 텍스트 나열 대신 카드 구획
- **충분한 여백 + 명확한 폰트 위계** = 깔끔함의 핵심
- Streamlit `.streamlit/config.toml` 로 테마 지정 + 커스텀 CSS

> 디자인 목업: `docs/mockup.html` (브라우저로 열어서 확인)

## 8. 폴더 구조 (과제 제출 요건 반영)

```
kw-oss-finalexam/
├─ front/
│  ├─ app.py              # Streamlit: 위저드 입력 + requests로 back 호출
│  ├─ Dockerfile
│  └─ requirements.txt    # streamlit, requests
├─ back/
│  ├─ main.py             # FastAPI: /recommend 엔드포인트
│  ├─ logic.py            # 규칙 스코어링 + Claude 호출 + fallback
│  ├─ Dockerfile
│  └─ requirements.txt    # fastapi, uvicorn, pydantic, anthropic, httpx
├─ docker-compose.yml     # 두 컨테이너 한번에 (EC2 배포 간편)
├─ .gitignore             # __pycache__, .env, venv 등
├─ .env.example           # ANTHROPIC_API_KEY, API_URL 템플릿
├─ docs/
│  └─ mockup.html         # 디자인 목업
├─ PLAN.md                # 본 기획서
└─ README.md              # 실행법 + EC2 배포 가이드 + 데모 설명
```

설계 포인트:
- Streamlit이 FastAPI 부를 주소를 **환경변수 `API_URL`** 로 → 로컬/EC2 둘 다 대응
- `ANTHROPIC_API_KEY` 는 `.env` 로 관리, `.gitignore` 처리 (키 노출 금지)

## 9. 배포 계획 (EC2)

1. 로컬에서 front/back/docker-compose 완성 + 동작 확인
2. EC2(Learner Lab) 인스턴스 생성 + 보안그룹 포트 개방(8501 streamlit, 8000 fastapi)
3. 코드 받아서 `docker compose up -d`
4. 외부에서 `http://<EC2-IP>:8501` 접속 → 동작 확인
5. 데모 영상 촬영
